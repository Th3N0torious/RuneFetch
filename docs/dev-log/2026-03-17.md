# Development Log

# Python Virtual Environment Setup

Moved into the backend directory:

cd backend

Created a Python virtual environment:

python -m venv venv

Explanation:

python
Runs the Python interpreter.

-m
Tells Python to execute a module.

venv
Python’s built-in module used to create isolated environments.

venv (second argument)
The name of the directory where the environment will be created.

Purpose:
Virtual environments isolate dependencies so they do not conflict with system Python or other projects.

Resulting structure:

backend
 ├ venv

Activated the environment:

Purpose:

When activated the terminal prompt changes to:

# Backend Dependency Installation

Installed required backend dependencies:

pip install django
pip install djangorestframework
pip install psycopg2-binary
pip install django-cors-headers
pip install python-dotenv
pip install requests

Purpose of each package:

Django
Primary Python web framework used to build the backend server.

Django REST Framework
Provides tools for building REST APIs and returning JSON responses.

psycopg2-binary
PostgreSQL database adapter for Python.

django-cors-headers
Allows cross-origin requests so the future React frontend can communicate with the Django backend.

python-dotenv
Allows configuration variables to be loaded from a .env file.

requests
HTTP client used to communicate with external APIs such as Wise Old Man.
